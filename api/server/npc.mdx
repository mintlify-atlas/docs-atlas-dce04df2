## Overview

The `Npc` struct represents a non-player character entity with AI behavior. NPCs can be hostile enemies, friendly quest givers, merchants, or passive creatures. They use a finite state machine for combat AI and wandering behavior.

## NPC State Machine

```rust
pub enum NpcState {
    Idle,       // Standing still, looking for players
    Chasing,    // Moving toward target player
    Attacking,  // In range of target, attacking
    Returning,  // Moving back to spawn point
    Dead,       // Dead, waiting to respawn
    Wandering,  // Random walk within wander_radius
}
```

### State Transitions

**Idle → Chasing:**
- Hostile NPC detects player within aggro_range

**Idle → Wandering:**
- Wander enabled and idle_until timer expired
- Picks random walkable target within wander_radius of spawn

**Chasing → Attacking:**
- Target is within attack_range (cardinal direction only)
- Movement cooldown complete

**Chasing → Returning:**
- NPC moved too far from spawn (> chase_range)
- Target moved too far away (> chase_range)
- Target died or disconnected

**Attacking → Chasing:**
- Target moved out of attack_range or not in cardinal direction

**Returning → Idle:**
- NPC reached spawn point

**Wandering → Idle:**
- Reached wander target
- Stuck for 4+ moves without progress

**Wandering → Chasing:**
- Hostile NPC detects player within aggro_range (interrupts wandering)

## Level Scaling

NPC stats scale with their level:

```rust
// HP scaling: +10% per level above 1
fn scale_hp(base_hp: i32, level: i32) -> i32 {
    let multiplier = 1.0 + 0.10 * (level - 1).max(0) as f64;
    (base_hp as f64 * multiplier).round() as i32
}

// Damage scaling: +15% per level above 1
fn scale_damage(base_damage: i32, level: i32) -> i32 {
    let multiplier = 1.0 + 0.15 * (level - 1).max(0) as f64;
    (base_damage as f64 * multiplier).round() as i32
}
```

**Example:**
- Level 1 Pig: 50 HP, 5 damage
- Level 5 Pig: 70 HP (~50 × 1.4), 8 damage (~5 × 1.6)

## AI Update Loop

```rust
pub fn update(
    &mut self,
    _delta: f32,
    players: &[(String, i32, i32, i32)],
    other_npc_positions: &HashSet<(i32, i32)>,
    current_time: u64,
    walkable_check: &dyn Fn(i32, i32) -> bool,
) -> Option<(String, i32)>
```

Updates NPC AI for one tick. Returns `Some((target_id, damage))` if the NPC attacks.

<ParamField path="players" type="&[(String, i32, i32, i32)]" required>
  List of (player_id, x, y, hp) for all players in range
</ParamField>

<ParamField path="other_npc_positions" type="&HashSet<(i32, i32)>" required>
  Occupied tile positions for collision avoidance
</ParamField>

<ParamField path="current_time" type="u64" required>
  Current timestamp in milliseconds
</ParamField>

<ParamField path="walkable_check" type="&dyn Fn(i32, i32) -> bool" required>
  Callback to check if a tile is walkable (no collision)
</ParamField>

<ResponseField name="attack" type="Option<(String, i32)>">
  If NPC attacks this tick, returns (target_player_id, damage)
</ResponseField>

### AI Behavior Details

**Idle State:**
1. If hostile and player in aggro_range → Chasing
2. If wander enabled and idle_until expired → Wandering

**Wandering State:**
1. If hostile and player in aggro_range → Chasing (interrupt)
2. Move toward wander_target
3. If reached target → Idle with random pause
4. If stuck (4 moves without progress) → Idle

**Chasing State:**
1. Check target still alive and in range
2. If too far from spawn or target → Returning
3. If in attack_range (cardinal only) and cooldown ready → Attacking
4. Otherwise, move toward target (one tile at a time)

**Attacking State:**
1. Face target
2. If movement and attack cooldowns ready → Deal damage
3. If target out of range → Chasing
4. If target lost → Returning

**Returning State:**
1. Move toward spawn point
2. If reached spawn → Idle

### respawn()

```rust
pub fn respawn(&mut self)
```

Respawns the NPC at its spawn point.

**Effects:**
- Teleports to (spawn_x, spawn_y)
- Restores HP to max_hp
- Sets state to Idle
- Resets facing to spawn_direction
- Clears target and wander state

## Example Usage

```rust
use std::collections::HashSet;

// Create NPC from prototype
let pig = Npc::from_prototype(
    "pig_001",
    "pig",
    &pig_prototype,
    100,
    50,
    5,      // Level 5
    Some("down"),
);

// Main game loop
loop {
    let current_time = get_current_time_ms();
    
    // Collect nearby players
    let players = vec![
        ("player_1".to_string(), 102, 52, 80),
        ("player_2".to_string(), 95, 48, 100),
    ];
    
    // Get occupied tiles for collision
    let mut occupied = HashSet::new();
    occupied.insert((101, 51));
    
    // Update AI
    if let Some((target_id, damage)) = pig.update(
        0.05,
        &players,
        &occupied,
        current_time,
        &|x, y| is_tile_walkable(x, y),
    ) {
        println!("Pig attacks {} for {} damage!", target_id, damage);
    }
    
    // Apply HP regen
    pig.apply_regen(current_time);
    
    // Check respawn
    if pig.state == NpcState::Dead && pig.ready_to_respawn(current_time) {
        pig.respawn();
        println!("Pig respawned!");
    }
}
```
