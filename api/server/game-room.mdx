## Overview

The `GameRoom` struct is the central server-side game engine that manages all game state, player sessions, NPCs, world data, and the main tick loop. It runs at 20 ticks per second and handles movement, combat, crafting, quests, and all game systems.

**Implementation Details:**
- Loads the world map from `maps/world_0`
- Preloads chunks around spawn point (radius: 3 chunks)
- Spawns all NPCs from chunk entity_spawns
- Initializes all game systems (gathering, woodcutting, mining, farming)
- Restores saved farming patches and plot unlocks from database
- Caches portal tile positions for NPC pathfinding

### Main Tick Loop

```rust
pub async fn tick(&self)
```

Runs the main game loop logic for one tick (called at 20 Hz).

**Processing Order:**
1. Movement tick - process pending player movements
2. Auto-action tick - handle OSRS-style click-to-act (attack/mine/chop)
3. NPC AI tick - update NPC state machines and attacks
4. Combat tick - resolve player attacks on NPCs/players
5. Resource gathering tick - fishing, woodcutting, mining
6. Crafting tick - check for completed timed crafts
7. HP/MP/Prayer regeneration
8. Quest objective updates
9. Farming crop growth
10. Shop restocking
11. Buff expiration
12. Ground item cleanup
13. State synchronization broadcast to clients

### Player Management

```rust
pub async fn add_player(&self, player: Player)
```

Adds a player to the game room.

<ParamField path="player" type="Player" required>
  The player entity to add
</ParamField>

**Effects:**
- Adds player to the players HashMap
- Updates player_chunks tracking
- Initializes delta sync state for bandwidth optimization
- Player becomes visible to other players in view distance

---

```rust
pub async fn remove_player(&self, player_id: &str)
```

Removes a player from the game room.

<ParamField path="player_id" type="&str" required>
  ID of the player to remove
</ParamField>

**Effects:**
- Removes player from players HashMap
- Frees their chair if sitting
- Clears any active trades
- Removes from player_chunks tracking
- Other players no longer see this player

### Player Spawning

```rust
pub async fn respawn_player(&self, player_id: &str)
```

Respawns a dead player at the world spawn point.

<ParamField path="player_id" type="&str" required>
  ID of the player to respawn
</ParamField>

**Effects:**
- Resets player position to `(WORLD_SPAWN_X, WORLD_SPAWN_Y)` = (15, 4)
- Restores HP, prayer points, and mana to maximum
- Clears auto-action and target
- Frees occupied chair if any
- Resets movement cooldown

### State Synchronization

```rust
pub async fn broadcast_state_sync(&self, full_sync: bool)
```

Broadcasts game state updates to all connected clients.

<ParamField path="full_sync" type="bool" required>
  If true, sends full state. If false, sends delta updates for bandwidth optimization.
</ParamField>

**StateSync Message Contents:**
- All players in view distance (VIEW_DISTANCE = 40 tiles)
- All NPCs in view distance
- All ground items in view distance
- Current server tick
- Instance-specific data if player is in an instance

**Optimization:**
- Delta sync sends only changed entities every tick
- Full sync forced every 20 ticks (1 second) to prevent desync
- Per-player culling based on Chebyshev distance

## Example Usage

```rust
use std::sync::Arc;

// Create game room
let game_room = GameRoom::new(
    "main",
    entity_registry,
    quest_registry,
    crafting_registry,
    item_registry,
    prayer_registry,
    player_instances,
    instance_manager,
    Some(db),
    interior_registry,
    chest_registry,
).await;

// Add player
let player = Player::new(
    "player_123",
    "Steve",
    15,
    4,
    "male",
    "tan",
    Some(2),
    Some(4),
);
game_room.add_player(player).await;

// Main game loop
loop {
    game_room.tick().await;
    tokio::time::sleep(tokio::time::Duration::from_millis(50)).await; // 20 Hz
}
```
