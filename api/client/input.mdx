## Overview

The `InputHandler` processes player input from keyboard, mouse, and touch controls, converting them into game commands. It manages click-to-move pathfinding, auto-actions, and UI interactions.

## InputHandler

Main input processing system.

### Methods

<ParamField path="process_input(state, audio)" type="Vec<InputCommand>">
  Process all input for the current frame and return commands to execute
  
  <ParamField path="state" type="&mut GameState" required>
    Game state (updated with path state, auto-actions, etc.)
  </ParamField>
  
  <ParamField path="audio" type="&mut AudioManager" required>
    Audio manager for UI sound effects
  </ParamField>
  
  Returns a list of `InputCommand` to send to the server.
</ParamField>

## Pathfinding

The input handler includes automatic pathfinding for click-to-move:

### Auto-Pathing

<ParamField path="auto_path" type="Option<PathState>">
  Current active path (stored in GameState)
  
  Contains:
  - `path`: Vec of (x, y) tiles to follow
  - `current_index`: Progress through path
  - `destination`: Final destination
  - `pickup_target`: Item to pick up at end
  - `interact_target`: NPC to talk to at end
</ParamField>

### Path Rebuilding

Paths are automatically rebuilt when:
- An entity moves (NPC, other player)
- A tile becomes blocked/unblocked
- An auto-action target moves

The system uses "optimistic splice" to preserve already-committed movement:

```rust
// Reuses first 1-6 tiles of old path if they're still valid
find_path_with_optimistic_splice(start, goal, occupied, max_distance)
```

## Keyboard Bindings

### Movement (Modern Controls)

- **W/A/S/D** - Move up/left/down/right
- **Space** - Dash ability

### Movement (Classic Controls)

- **Arrow Keys** - Move in 4 directions
- **Ctrl** - Attack/interact

### Actions

- **Left Click** - Move, interact, attack (context-sensitive)
- **Right Click** - Open context menu
- **Shift + Click** - Drop item (if enabled)
- **1-9** - Hotkey slots

### UI

- **Enter** - Open chat (modern) or send message (classic)
- **Escape** - Close panels
- **I** - Toggle inventory
- **C** - Toggle character panel
- **P** - Toggle prayer/spell book

## Example Usage

```rust
use aeven_client::input::InputHandler;

let mut input_handler = InputHandler::new();

// Process input each frame
let commands = input_handler.process_input(&mut game_state, &mut audio);

// Send commands to server
for command in commands {
    match command {
        InputCommand::Move { dx, dy } => {
            network.send(&ClientMessage::Move { dx, dy, seq: 0 });
        }
        InputCommand::Attack => {
            network.send(&ClientMessage::Attack);
        }
        InputCommand::Chat { text, channel } => {
            network.send(&ClientMessage::Chat { text, channel });
        }
        _ => {}
    }
}
```

## Auto-Action System

The auto-action system provides OSRS-style click-to-act behavior:

1. **Click entity** → Pathfind to range → Start action when in range
2. **Entity moves** → Update path to follow
3. **Action completes** → Continue if entity still valid
4. **Target dies/despawns** → Cancel action

```rust
// Stored in GameState
pub struct AutoActionState {
    pub target_type: String,  // "npc", "player", "resource"
    pub target_id: String,    // Entity ID or "x,y,gid"
    pub action: String,       // "attack", "chop", "mine"
    pub confirmed: bool,      // Server acknowledged
}
```
