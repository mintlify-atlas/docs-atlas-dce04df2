## Overview

The `NetworkClient` handles all network communication between the client and server using WebSockets. It manages connection lifecycle, message encoding/decoding, and automatic reconnection.

## NetworkClient

WebSocket client with matchmaking and reconnection support.

## Connection Lifecycle

The client manages connection through several states:

1. **Disconnected** - No connection to server
2. **Matchmaking** - Requesting room assignment via HTTP
3. **Connecting** - Establishing WebSocket connection
4. **Connected** - Active connection, sending/receiving messages

### Reconnection

The client automatically attempts to reconnect after disconnection:

- Maximum 3 reconnection attempts
- 2 second delay between attempts
- Preserves authentication token and character ID
- Spectator mode uses separate reconnection path

## Message Processing

The `poll()` method processes incoming messages in this order:

1. **Pong messages** - Handled immediately for accurate latency measurement
2. **State sync messages** - Batched and processed in tick order (max 4 per frame)
3. **Other messages** - Processed immediately

State sync messages older than the current server tick are discarded to prevent stale updates.

## Example Usage

```rust
use aeven_client::network::{NetworkClient, ClientMessage};

// Create authenticated client
let mut client = NetworkClient::new_authenticated(
    "ws://localhost:2567",
    "jwt_token_here",
    12345,
);

// In main loop
loop {
    // Process network messages
    client.poll(&mut game_state);
    
    // Check connection status
    if client.is_connected() {
        // Send movement
        client.send(&ClientMessage::Move {
            dx: 1.0,
            dy: 0.0,
            seq: 0,
        });
        
        // Send chat
        client.send(&ClientMessage::Chat {
            text: "Hello world!".to_string(),
            channel: "public".to_string(),
        });
    }
}
```

## Platform Differences

### WebAssembly (WASM)

On WASM, matchmaking is performed by JavaScript before loading the game:

```javascript
// JavaScript matchmaking
const response = await fetch('/matchmake/joinOrCreate/game_room', {
    method: 'POST',
    headers: { 'Authorization': 'Bearer ' + token },
    body: JSON.stringify({ characterId: 12345 })
});
const { room, sessionToken } = await response.json();
localStorage.setItem('roomId', room.roomId);
localStorage.setItem('sessionToken', sessionToken);
```

The Rust client then reads these values from localStorage.

### Desktop

On desktop, the client performs matchmaking directly via HTTP before opening the WebSocket.
