## Overview

The `Renderer` handles all game rendering using an isometric projection. It manages sprite atlases, performs depth sorting for proper layering, and renders the game world, entities, and UI.

Main rendering system with sprite atlas management.

## Depth Sorting

The renderer uses a painter's algorithm for proper layering:

1. **Layer-based ordering** - Ground tiles < Entities < Effects (multiplied by 10000)
2. **Within-layer sorting** - By `x + y` coordinate (down-right renders on top)
3. **Rendering order** - Lowest depth first, highest depth last

This ensures entities further down and to the right appear in front of those up and to the left.

## Example Usage

```rust
use aeven_client::render::{Renderer, world_to_screen, screen_to_world, calculate_depth};

// Initialize renderer
let mut renderer = Renderer::new(&mut audio).await;

// Convert coordinates
let (screen_x, screen_y) = world_to_screen(10.0, 5.0, &camera);
let (world_x, world_y) = screen_to_world(mouse_x, mouse_y, &camera);

// Calculate depth for sorting
let player_depth = calculate_depth(player.x, player.y, 1);
let tree_depth = calculate_depth(tree_x as f32, tree_y as f32, 0);

if tree_depth < player_depth {
    // Tree renders first (behind player)
}

// Access sprites
if let Some((texture, rect)) = renderer.item_sprites.get("iron_sword") {
    // Draw item sprite
}

if let Some((sheet, offset)) = renderer.player_sprites.get("male_tan") {
    // Draw player animation frame
}
```

## Rendering Pipeline

The renderer processes each frame in this order:

1. **Ground Layer** - Tiles, water, walls
2. **Object Layer** - Trees, rocks, buildings (sorted by depth)
3. **Entity Layer** - Players, NPCs, ground items (sorted by depth)
4. **Overhead Layer** - Name tags, health bars, damage numbers
5. **Effect Layer** - Particles, spell effects (sorted by depth)
6. **UI Layer** - Panels, inventory, chat

## Performance Optimizations

- **Sprite atlases** on mobile reduce texture switches
- **Pixel snapping** prevents sub-pixel jitter
- **Depth pre-sorting** minimizes overdraw
- **Culling** for off-screen entities
- **Cached text rendering** for UI
- **Instanced rendering** for tiles
