Aeven uses MessagePack for efficient binary serialization of messages between client and server. All messages follow a tagged enum structure with a `type` field for message identification.

## Message Format

Messages are encoded as MessagePack arrays in the Colyseus protocol format:

```rust
[protocol_code, message_type, message_data]
```

### Protocol Codes

<ResponseField name="protocol_code" type="u8">
  Colyseus protocol identifier
  
  - `9` - Handshake
  - `11` - Error
  - `13` - RoomData (standard game messages)
  - `14` - RoomState (full state snapshot)
  - `15` - RoomStatePatch (state delta)
</ResponseField>

### Compression

Messages support optional deflate compression with a prefix byte:

- `0x00` - Uncompressed MessagePack data follows
- `0x01` - Deflate-compressed MessagePack data follows
- Other - Legacy uncompressed (no prefix)

See `client/src/network/protocol.rs:35` for decompression implementation.

## Example Messages

### Client Move Message

```json
{
  "type": "move",
  "dx": 1.0,
  "dy": 0.0,
  "seq": 42
}
```

Encoded as MessagePack (hex):
```
93 0D A4 6D 6F 76 65 83 A2 64 78 CB 3F F0 00 00 00 00 00 00
A2 64 79 00 A3 73 65 71 2A
```

Breakdown:
- `93` - Array with 3 elements
- `0D` - Protocol code 13 (RoomData)
- `A4 6D 6F 76 65` - String "move"
- `83` - Map with 3 entries (dx, dy, seq)

### Server Welcome Message

```json
{
  "player_id": "char_12345",
  "is_new_character": false
}
```

Sent as MessagePack array:
```
[13, "welcome", {"player_id": "char_12345", "is_new_character": false}]
```
